# 特に見ていただきたい点

- Bootstrapとメディアクエリを活用したレスポンシブ対応
- jQueryを用いた動的アニメーション/ajax通信の実装(インクリメンタルサーチ/メッセージ自動更新)
- 外部APIを利用したログイン機能
- インフラ周り(AWSを使用/Capistranoによる自動デプロイ/ELBをエンドポイントとする常時SSL化/画像データ保存でクラウドストレージS3を使用)

# 機能ならびに使用技術

- フロントエンド
  - Haml/Sassでのマークアップ
  - jQueryを用いた動的アニメーション/ajax通信の実装
  - Bootstrapとメディアクエリを活用したレスポンシブ対応

- サーバーサイド
  - ユーザー登録(/編集)機能(gem Deviseを使用)
  - ユーザー画像の登録機能 (gem Carrierwaveを使用)
  - Twitter/Facebook/Googleアカウントでのログイン機能(gem OmniAuthを利用)
  - 学習内容投稿(/削除)機能
  - 学習時間の可視化(gem Chartkickを使用)
  - 相互フォローユーザーとのメッセージ機能
  - ユーザー検索機能(ajax通信での非同期検索(インクリメンタルサーチ))
  - 他ユーザーへのフォロー機能
  - いいね機能
  - 不適切用語のフィルター(gem Antivirusを使用)
  - 単体テスト(RSpec)
  - 統合テスト(外部APIを利用した機能のテスト(モックの使用),gem selenium-webdriver/ChromeDriverを用いたブラウザテスト)

- インフラ
  - CapistranoによるEC2インスタンスへの自動デプロイ
  - お名前.comで独自ドメインを取得後、Route53/ELB/Certificate Managerを用いて常時SSL化
  - S3を用いた画像データのクラウドストレージ保存

# クラウドアーキテクチャ

![go-enインフラ](https://user-images.githubusercontent.com/53807858/67618575-31d80180-f82c-11e9-9449-4d9d95dd5c27.png)

EC2インスタンスにCapistranoを用いて、ローカル環境からコマンド一つで自動デプロイを行うようにしています。
またCertificate ManagerでSSLサーバ証明書を発行し、お名前.comで取得したドメインをRoute53/ELBで常時SSL化しています。

クラウドストレージにはS3を採用しました。

GitHub上では可視化されませんが、AWSではVPC内のセキュリティグループやIAMユーザーポリシーなどの設定を行い、実際の現場でのアーキテクチャを想定して構成されています。

尚、アプリ内の秘匿情報等はRailsのseacretsを活用し、keyとidを本番環境の環境変数に登録することで管理しています。

# Go-Enを作った理由

### ①駆け出しのエンジニア同士をつなげるサービスを作りたかったからです。

私は2019年7月からプログラミングスクールに通いはじめました。もちろんWEB系の言語はほとんど使用したことなく(progateを１週した程度)、新卒で入社した会社を退職したせいもあってか、「こんな難しいことを仕事にしていけるかな？」「このまま勉強をしていていいのかな？」といった不安が少しばかりありました。

しかし、この不安な気持ちはすぐになくなりました。志の高い駆け出しエンジニアがスクールにはたくさんいたためです。技術もモチベーションも刺激しあえるたくさんの駆け出しエンジニアに出会えたことは「スクールに入ってよかった」と思える要因の一つです。「この人には負けない」「こんなモダンな技術使っている人もいる」といった感情が私の成長を後押ししてくれたのは確かですし、実際に独学していた4ヶ月前とは比べ物にならないくらい技術力は向上したと思います。


独学でプログラミング学習を始めた人の9割以上が挫折すると聞いたことがあります。正直、気持ちはわかります。新しいことに挑戦するときはワクワクするけど、やっぱり不安もありますし、くじけそうになることもあります。これからプログラミング学習をしていきたい全ての方たちに、Go-En(ご縁)を使ってたくさんの駆け出しエンジニアと繋がって欲しい、モチベーションを高めて欲しいと思い、このアプリケーションを作成しました。

### ②アウトプットを簡単にできるサービスを作りたかったからです。

エンジニアにとってアウトプットはとても重要です。本当に重要です。

私は初め、この意味がわかりませんでした。「みんなアウトプットは大事っていうけどなんで？」「ていうかなにからはじめればいいの？」と思っていました。しかし、"物は試しだ"と思い、見切り発車的にアウトプットを続けていました。[現在も継続してつづけています。](https://qiita.com/nakanishi03)そして、その重要性がわかるようになってきました。

例えば、間違っていることを指摘してもらえることで理解度が向上したり、自分の発信が誰かのためになるとモチベーションが上がったりします。最近では、仕事をいただけるようにもなりました。私の記事を見たある経営者の方が、SNSを通じて仕事を依頼してくれました。現在その経営者が運用しているサービスにジョインさせていただき、テストコードを書いています。初学者こそアウトプットをすべきだと強く思うようになりました。

プログラミング初学者に簡単に日々のアウトプットをしてほしいと思い、このサービスを作りました。「Qiitaで記事を書くのはちょっと怖い」「Twitterはエンジニア以外の人にもたくさんフォローされてるから嫌だ」そんな方にGo-Enを使ってアウトプットの習慣をつけていただきたいです。
